# 015. 复权因子单测补全 卡

**状态**: `closed`
**类型**: `data / testing / adjust-factor`
**模块**: `data`

## 1. 定位

这张卡只解决一件事：

补齐复权因子计算链的最小必要单元测试，确保 `014` 一旦发现问题时，能够用稳定、离线、可复跑的测试快速定位根因。

## 2. 固定因素

1. 本卡只做单测补齐，不改 data 主线来源路线。
2. 所有测试必须使用固定输入数据，不依赖正式数据库、不依赖网络。
3. 测试覆盖要服务于正式主线最关键的权益事件，而不是追求表面数量。
4. 如果当前函数签名不稳定，必须先在记录中写清测试锚点，再补用例。

## 3. 输出要求

1. 明确待测函数与核心计算路径。
2. 覆盖分红、送股、配股、复合事件与边界条件。
3. 测试命名、样例数据、断言口径必须可复述。
4. 形成 evidence / record / conclusion 四件套。

### 3.1 当前正式补测锚点

1. 目标函数：`src/lq/data/compute/adjust.py` 中的 `compute_backward_factors()` 与 `apply_backward_adjustment()`。
2. 当前仓库中未确认现成的 `adjust_factor` 单测文件，因此本卡第一步是**新建测试文件**，不是补跑旧测试。
3. 按当前测试目录规范，优先落点应为 `tests/unit/data/` 下的新测试文件。

## 4. 本卡回答的问题

1. 哪些复权因子计算路径最需要最小稳定测试保护？
2. `014` 若发现抽样异常，是否可以用这些单测快速复现？
3. 当前复权因子实现是否还缺关键边界测试？

## 5. 非目标

1. 不在本卡重写完整 data 模块测试体系。
2. 不在本卡做数据库集成测试替代。
3. 不以“凑到多少用例”为目标替代关键路径覆盖。

## 6. 证据目标

1. 新增测试文件与测试用例清单。
2. pytest 运行证据。
3. 对关键权益事件覆盖范围的说明。

## 7. 关闭条件

1. 最小关键路径测试已经补齐并可复跑。
2. evidence / record / conclusion 全部补齐。
3. 结论能明确回答“复权因子链是否已具备最小单测保护”。
