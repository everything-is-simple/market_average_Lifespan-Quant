# 04-reference 目录入口

## 当前状态

`docs/04-reference/` 已经完成私有化清理。

从 `2026-03-26` 起，这个目录不再在仓库内保存原始参考资料、供应商交付物、授权原件、教程视频、凭证材料或第三方原始表格。

保留这份入口文件的目的只有一个：

说明这些资料为什么不再放在仓库里，以及现在应去哪里找。

## 为什么迁出仓库

原 `04-reference` 下的内容混合了三类材料：

1. 可公开查到的市场规则摘要。
2. 第三方供应商资料、通道说明与教程素材。
3. 受授权约束或不适合公开再分发的原始表格、压缩包与附件。

这些材料继续放在仓库里，会带来三个问题：

1. 容易被误当成正式系统口径。
2. 容易把凭证、网关、供应商交付物一起带进版本库。
3. 容易让受授权约束的原始资料进入不必要的传播范围。

因此当前口径改为：

1. 仓库只保留入口说明。
2. 人读参考文档转移到私有 `Report` 目录。
3. 原始数据资料与授权文件转移到私有 `Data` 目录。

2. 压缩包

## 当前使用方式

如果你只是想确认系统正式口径，仍然不要先看这里，而要先看：

1. `docs/01-design/`
2. `docs/02-spec/`
3. `docs/03-execution/`

只有在需要补外部背景、供应商依据或第三方规则时，才进入私有参考目录查阅。

## 使用纪律

1. 私有参考资料不能替代正式 `design / spec / conclusion`。
2. 凭证、token、网关信息不得重新放回仓库。
3. 受授权约束的原始附件只保留在私有目录，不进入公共版本库。
4. 若外部资料影响系统正式口径，应回写到正式文档，而不是只留在私有参考区。

## 例外：仓库内保留的系统参考总图

尽管 `04-reference` 不再保存原始第三方资料，但仍允许保留少量**仓库自写的系统参考总图**，前提是它们：

1. 不包含受授权约束的原始外部资料。
2. 不替代正式 `design / spec / conclusion`。
3. 明确标注自己只是参考导航，不是正式业务裁决。

当前保留的系统参考总图包括：

1. `system-selected-concepts-and-implementation-boundary-20260328.md`
2. `README-当前系统最值得吸收的未来研究方向-20260329.md`
