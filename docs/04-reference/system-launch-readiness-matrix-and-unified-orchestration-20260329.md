# 系统正式上线成熟度总表与统一编排图 / 2026-03-29

## 1. 这份文档解决什么问题

这份文档不新增业务裁决，也不替代 `docs/03-execution/` 下的正式结论。

它只回答一个问题：

`截至 2026-03-29，我们的系统到底已经完成到哪一层，哪些能力可以信，哪些还只能带边界地用，哪些上线前必须补，哪些虽然重要但当前不该卡住正式上线。`

## 2. 先把“上线”拆成三层

### 2.1 研究验证正式上线

人话：
系统已经能稳定跑研究主线、做正式验证、做正式治理闭环。

边界：
还不能说“普通散户拿真实历史价格照着做就差不多”。

### 2.2 执行仿真正式上线

人话：
系统不仅能做研究验证，还已经把执行价口径和数据可信度审到更接近真实交易。

边界：
仍然不等于已经把完整交易业务层、团队运营层都搭好。

### 2.3 零售可执行正式上线

人话：
系统已经能把研究逻辑、执行逻辑和交易者可理解的动作合同连成一套能落地的话。

边界：
截至当前，这一层还不能诚实宣布已完成。

## 3. 信任分级

1. `T1 / 核心可信`
   当前正式主线能直接站住的能力。
2. `T2 / 有边界可用`
   已正式存在，但必须写清边界，不能过度外推。
3. `T3 / 部分承接`
   系统吸收了其中一部分，但还不是正式上线承诺。
4. `T4 / 研究储备`
   还只是想法、命题、阅读沉淀或未来方向。

## 7. 统一编排顺序

### 7.1 现在就该承认的上线层级

最诚实的说法是：

`当前系统已经非常接近“研究验证正式上线”，但还差 131 的正式收口；距离“执行仿真正式上线”还差 raw-execution 对齐和 data 可信度重验；距离“零售可执行正式上线”则还差一整层执行合同与业务层整理。`

### 7.2 建议队列

1. 先关 `131`
   - 把 PAS 五 trigger 研究层地位、准入和 alpha 合同收死。
2. 再开 `raw-execution` 对齐卡
   - 把研究层价格口径和更接近真实执行的口径拆干净。
3. 再开 `data` 可信度重验卡
   - 把执行层最底座补硬。
4. 然后再决定是否进入“执行仿真正式上线”宣布阶段。
5. 零售可执行层相关话题，在上述两卡未完成前一律不提前宣布。

## 8. 给普通看 K 线交易者的人话

这套系统现在最稳的，不是“某一个神奇图形”，而是“先看市场背景，再看形态，再看怎么进退”。已经跑出来的主线，足够告诉我们哪些招式靠谱、哪些只能挑环境用、哪些先别碰；但它还没到“你拿着历史价格照抄就能安全活下去”的程度。

## 9. 给普通交易员的合同话

当前系统可正式承诺的是 `research_validation_ready`，不是 `retail_execution_ready`。上线编排必须先完成 PAS 五 trigger 的研究层合同化，再补 `raw-execution` 对齐与 `data` 可信度重验；在这之前，一切“可真实执行”表述都属于越权。
