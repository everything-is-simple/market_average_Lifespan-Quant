# 当前系统最值得吸收的未来研究方向 / 2026-03-29

## 1. 这份文档解决什么问题

这份文档不做新的正式业务裁决，也不改写现行 `design / spec / conclusion`。

它只回答一个问题：

`如果站在 2026-03-29 当前系统开发进度上，从长期阅读沉淀与价格行为/系统交易资料中挑“最值得进入正式研究池”的概念，优先级应该怎样排。`

这是一份：

`研究导航 / 概念优先级参考`

而不是：

`正式准入清单`

## 3. 判定规则

### 3.1 优先级 A

同时满足以下条件中的多数：

1. 与当前 `MALF -> PAS -> position -> trade -> system` 主线强相关。
2. 已在现行口径中被明确记为“部分承担但不是完整实现”，或者离正式 runner 很近。
3. 可以被压成稳定合同、字段、枚举、detector 或验证卡，而不是只停留在书义。
4. 能直接提高正式 trigger 语义、市场环境分型、过滤器或执行层 readout 质量。

### 3.2 优先级 B

满足以下任一情况：

1. 很有价值，但仍需要更长一段语义收敛。
2. 与主线相关，但更偏 sidecar、扩展层或执行细化层。
3. 当前还不适合直接进入正式验证 runner。

### 3.3 优先级 C

满足以下任一情况：

1. 过于局部、标签化、形态化。
2. 与当前 A 股日线主线的正式治理距离较远。
3. 更适合留在阅读训练层，而不是进入正式系统概念层。

## 4. 优先级 A：最值得优先进入正式研究池

### A1. 关键位 / 支撑阻力 / 波段点 / 结构位

状态判断：

`最优先`

原因：

1. 当前系统已经在多个 trigger 内部局部使用了 support / support_band / hold 等语义。
2. 但这些仍是局部 detector 语言，缺少统一系统级合同。
3. 这是把价格行为阅读沉淀转换为系统语义的最好入口。

若未来推进，建议优先研究：

1. 水平关键位
2. 波段高低点
3. 测试点 / 回踩点
4. 结构突破后形成的新支撑 / 新阻力

一句话收口：

**先统一“结构位”，比先扩充形态名词更值钱。**

### A2. 突破 / 假突破 / 测试 / 回踩确认

状态判断：

`最优先`

原因：

1. 当前正式 trigger 已经天然围绕这组语义展开。
2. `BOF / PB / BPB / TST / CPB` 本质上都属于 breakout-family 的不同语义切片。
3. 如果不先把“突破失败学”讲清楚，后续 trigger 很容易永远停在 detector 名字层。

这一组最值得正式研究的，不是某个形态名，而是：

1. 什么叫有效突破
2. 什么叫失败突破
3. 什么叫测试
4. 什么叫确认
5. 什么叫“突破之后不应再回到哪里”

### A3. 二次入场 / 二次突破 / 第一 PB 假说

状态判断：

`最优先`

原因：

1. 这组概念在阅读材料中出现频率极高，而且与当前系统 trigger 语义天然相连。
2. `TST`、`CPB`、以及“上涨中的第一个 `PB` 最安全”都属于这一簇。
3. `125` 已经把“第一 PB”明确列为下一批有价值的独立执行线。

所以这条线不是泛概念，而是已经非常接近正式执行线。

### A4. 不利市场条件 / 不做过滤器

状态判断：

`最优先`

原因：

1. 当前系统已经有市场背景与顺逆关系，但“什么情况下不该做”还没有被正式做成统一过滤器语言。
2. 这类概念比“多找一个入场形态”更能提高主线质量。
3. 多数成熟交易书真正值钱的地方，最后都落在“不做什么”。

建议未来优先研究的内容：

1. 压缩且无方向
2. 结构混乱
3. 空间不足
4. 多重信号冲突
5. 背景不支持

### A5. 交易管理模板

状态判断：

`最优先`

原因：

1. 当前主线已经有 `1R / partial exit / trailing stop / fixed notional`，但这还是“基础执行合同”。
2. 未来如果要让系统更像系统，必须把“入场后如何演化”组织成正式模板。
3. 这是把 Lance / Bob / 许佳冲 / Al Brooks 的实操沉淀吸进系统的最稳入口之一。

建议未来优先研究的管理概念：

1. 初始止损
2. 保护性提损
3. 第一目标实现后的处理
4. 分批减仓
5. 剩余仓位跟踪

## 5. 优先级 B：很有价值，但不宜抢在最前面

### B1. YTC 架构区 / 架构定位规则

原因：

1. `125` 已经明确把 `YTC` 架构记为“已部分承担，但不是完整实现”。
2. 它很强，但直接整体迁入风险很高。
3. 更稳妥的做法是先把其中能落地的结构位、过滤器、交易过程拆出来，再反推架构区。

### B2. 强弱 / 顺强逆弱 / 相对强度

原因：

1. 当前系统已经有月线状态、顺逆流、trigger strength 等近似表达。
2. 但“强弱”还没有被正式统一成系统级合同。
3. 相对强度目前仍明确属于“当前未正式实现”。

这条线有价值，但不宜先抢在结构位与 breakout-family 前面。

### B3. 压缩 / 扩张 / 波动环境

原因：

1. 这组概念对“什么时候不做、什么时候期待突破”很重要。
2. 但它比结构位更容易滑向语义发散。
3. 更适合作为 A4 的补强层，而不是独立先行。

### B4. 分批入场 / 金字塔加仓 / 保护性减仓

原因：

1. 很实战。
2. 但如果基础 trigger 与过滤器语义还没收稳，先做这层容易把系统推回“战术花活”。
3. 它更适合放在交易管理模板之后细化。

## 6. 优先级 C：当前不建议优先进入正式系统概念层

### C1. 单根 K 线标签化名词

例如：

1. Pinbar
2. 单根反转棒的过细命名
3. 过多局部蜡烛形态标签

原因：

1. 可读性强，但系统价值不高。
2. 很容易导致“标签比结构重要”的错位。

### C2. 过细的超短盘口节奏

例如：

1. Bob 式极细微 intraday 处理
2. 对 tick / 1分钟 / 3点后微节奏的过度依赖

原因：

1. 当前系统正式主线是 A 股日线。
2. 这类语义太靠近特定市场微结构，不宜抢先进入正式层。

### C3. Al Brooks 大量原典术语整包迁入

原因：

1. 原典价值极高。
2. 但如果整包搬运，会极大提高系统语言复杂度。
3. 当前更适合“定点吸收”，不适合“术语整包准入”。

### C4. 宏观大理论整包系统化

原因：

1. 阅读价值高。
2. 但当前主线最缺的不是更大的理论，而是把已有结构语言冻结成合同。

## 7. 如果今天只排一个最小研究顺序

我建议当前系统未来研究顺序压成：

1. `结构位统一合同`
2. `突破 / 假突破 / 测试 / 回踩确认`
3. `二次入场 / 第一 PB / TST / CPB`
4. `不利市场条件 / 不做过滤器`
5. `交易管理模板`
6. `YTC 架构区补充收敛`
7. `强弱 / 相对强度`

## 8. 一句话总裁决

如果站在当前系统开发进度看，最值得吸进正式研究池的，不是更多形态名词，而是：

**结构位、突破失败学、二次入场、不做过滤器、以及交易管理模板。**

它们最有希望从长期阅读沉淀，真正长成下一批能落正式卡、能跑正式验证、能进入主线 runner 的系统能力。
