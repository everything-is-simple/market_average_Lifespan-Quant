# MALF 三层分工设计 / 2026-03-31（重构版）

> 继承来源：父系统 `10-malf-full-cycle-layering-20260324.md` + `12-malf-monthly-weekly-coordination-20260324.md`

## 1. 设计目的

**月线、周线、日线各自只负责一个问题，不交叉承担。**

这是 MALF 流水线的分层基础。

## 2. 三层分工

### 第一层：月线 → 长期背景

月线只回答：**这只股票现在处在什么长期背景里？**

| 层级 | 字段 | 含义 |
|------|------|------|
| 计算层 | `monthly_state_8` | 八态（形成/持续/衰竭/反转 × 牛/熊），内部阶段诊断 |
| 执行层 | `long_background_2` | `BULL / BEAR` 二分，由八态收敛 |

月线**不负责**顺逆关系，**不负责**触发信号。

### 第二层：周线 → 中期顺逆

周线只回答：**当前中期运动，相对月线背景，是顺着走还是逆着走？**

| 层级 | 字段 | 含义 |
|------|------|------|
| 计算层 | `weekly_flow` | `with_flow / against_flow` |
| 执行层 | `intermediate_role_2` | `MAINSTREAM / COUNTERTREND` |

周线**不负责**长期背景定义，**不负责**触发信号。

### 第三层：日线 → 新价结构

日线只回答：**这段趋势的新价活力怎样？新高/新低还在出现吗？间距有没有放大？**

| 维度 | 含义 |
|------|------|
| 新高/新低次数 | 窗口内创新价的交易日数量 |
| 新价间距 | 距上一次新价日的交易日间距（放大 = 衰竭信号） |

日线**不负责**长期背景或中期顺逆归属。

### PAS 触发器 → 不在 MALF 三层内

触发器和日线节奏是**两件事**。PAS 五触发器（BOF / PB / TST / CPB / BPB）由 `alpha/pas` 模块独立检测，消费 MALF 输出作为背景输入，不属于 MALF 计算范围。

## 3. 为什么这样分层

| 层级 | 理由 |
|------|------|
| 月线做背景 | A 股暴涨暴跌容易切碎周线长期层；月线收盘线对大背景更稳定 |
| 周线做顺逆 | 周线波段数量约为月线的三倍，最适合回答"主流还是逆流" |
| 日线做新价结构 | 日线最值钱的是节奏：新价密集 = 趋势健康；新价稀疏 = 趋势衰竭 |

类比：
- **月线 = 季节**（决定你默认穿什么衣服）
- **周线 = 天气**（决定今天出门带不带伞）
- **日线 = 脉搏**（告诉你趋势此刻的生命力）

## 4. 阅读顺序

```text
1. long_background_2   → 长期背景牛还是熊？
2. intermediate_role_2  → 当前中级波段顺势还是逆势？
3. malf_context_4       → 落在四格哪一格？
4. 三轴原始排位         → 波幅/时间/新价 在历史同类中排第几？
5. lifecycle_rank_*     → 三轴相加，趋势生命走到哪里了？
6. 日线新价节奏         → 新价日还在密集出现吗？
```

## 5. 与旧口径的纠偏

| 旧口径 | 当前正式口径 |
|--------|-------------|
| 长期趋势看周线 | 月线才是长期背景主线 |
| 中期关系固定为日线 | 中期关系由周线承担 |
| 日线承担中级主对象 | 日线只做新价结构；PAS 触发器由 `alpha/pas` 独立承担 |
