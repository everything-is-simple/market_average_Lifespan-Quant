# MALF 两步分工设计 / 2026-04-07（重构版）

> 继承来源：父系统 `10-malf-full-cycle-layering-20260324.md` + `12-malf-monthly-weekly-coordination-20260324.md`

## 1. 设计目的

MALF 回答一个问题：**当前趋势的生命走到哪里了？**

回答分两步，对应章程 §3：

1. **上下文分类**：月线 + 周线 → `malf_context_4`（四格之一）
2. **生命周期排位**：在同一四格内，用三轴度量当前波段在历史同类中的位置

两步不可分割——上下文让排位有意义，排位是系统的核心输出。

## 2. 第一步：上下文分类

### 月线 → 长期背景

月线只回答：**这只股票现在处在什么长期背景里？**

| 字段 | 含义 |
|------|------|
| `long_background_2` | `BULL / BEAR` |

### 周线 → 中期顺逆

周线只回答：**当前中期运动，相对月线背景，是顺着走还是逆着走？**

| 字段 | 含义 |
|------|------|
| `intermediate_role_2` | `MAINSTREAM / COUNTERTREND` |

### 组合 → 四格上下文

月线 + 周线组合产出 `malf_context_4`（`BULL_MAINSTREAM` / `BULL_COUNTERTREND` / `BEAR_MAINSTREAM` / `BEAR_COUNTERTREND`）。

月线**不负责**顺逆关系，周线**不负责**长期背景定义，两者都**不负责**触发信号。

## 3. 第二步：生命周期三轴度量

四格确定后，在同一格内用三把尺子度量当前中级波段：

| 轴 | 度量 | 回答 |
|----|------|------|
| 波动幅度 | 当前波段已走出的价格振幅 | 走了多远 |
| 时间跨度 | 当前波段已持续的交易时间 | 走了多久 |
| 新价结构 | 创新高/新低的次数与间隔 | 推进节奏如何 |

每轴产出原始历史排位区间 → 三轴相加得到总生命区间 → 四分位压缩辅助执行。

**PAS 触发器不属于 MALF**。PAS 五触发器（BOF / PB / TST / CPB / BPB）由 `alpha/pas` 模块独立检测，消费 MALF 输出作为背景输入。

## 4. 为什么这样分工

| 分工 | 理由 |
|------|------|
| 月线做背景 | A 股暴涨暴跌容易切碎周线长期层；月线收盘线对大背景更稳定 |
| 周线做顺逆 | 周线波段数量约为月线的三倍，最适合回答"主流还是逆流" |
| 三轴度量在四格内 | 只有同类波段可比，先分类再排位避免混排 |

## 5. 阅读顺序

```text
1. long_background_2   → 长期背景牛还是熊？
2. intermediate_role_2  → 当前中级波段顺势还是逆势？
3. malf_context_4       → 落在四格哪一格？
4. 三轴原始排位         → 波幅/时间/新价 在历史同类中排第几？
5. lifecycle_rank_*     → 三轴相加，趋势生命走到哪里了？
```

## 6. 与旧口径的纠偏

| 旧口径 | 当前正式口径 |
|--------|-------------|
| 月线/周线/日线三层并列 | 两步：上下文分类（月+周） → 三轴度量（波幅/时间/新价） |
| 长期趋势看周线 | 月线才是长期背景主线 |
| 中期关系固定为日线 | 中期关系由周线承担 |
| 日线是独立第三层 | 新价结构是三轴之一，在四格确定后才度量 |

### 6.1 已放弃设计的地位

1. `monthly_state_8` 与 `weekly_flow` 可以继续作为计算层诊断或兼容输入存在，但不得再被解释为 `MALF` 的执行主轴。
2. “月线 × 周线 × PAS”的三层矩阵，是旧时代对背景与触发器组织方式的描述，不是对中级波段生命本体的定义。
3. 任何把日线当作独立第三层、或把四分位当主读数的写法，均视为旧设计残留。
4. 当前正确口径固定为：`月线 + 周线先完成四格分类`，随后在同一格内对 `波幅 / 时间 / 新价结构` 做原始历史排位。
