## 1. 设计目的

本文冻结一件事：

**月线、周线、日线各自只负责一个问题，不交叉承担。**

这是当前系统主线的分层基础，不允许跨层混用口径。

## 2. 三层分工（冻结）

### 第一层：月线 → 长期背景

月线只回答：

> 这只股票现在处在什么长期背景状态里？

计算层字段：`monthly_state_8`（八态，用于内部阶段诊断）
执行层字段：`long_background_2`（`BULL / BEAR` 二分，由八态收敛）

职责边界：
1. 长期偏多还是偏空
2. 长期背景是否已经转折
3. 当前处于形成、持续、衰竭还是反转阶段（计算层细分）

月线**不负责**顺逆关系，**不负责**触发信号。

### 第二层：周线 → 中期顺逆关系

周线只回答：

> 当前中期运动，相对月线长期背景，是顺着走还是逆着走？

计算层字段：`weekly_flow_relation_to_monthly`（取值 `with_flow / against_flow`）
执行层字段：`intermediate_role_2`（`MAINSTREAM / COUNTERTREND`，由 weekly_flow 映射）

职责边界：
1. 当前周线这段是顺势推进
2. 还是逆势回调/反弹

周线**不负责**长期背景定义，**不负责**触发信号。

### 第三层：日线 → 节奏现象（MALF 负责）

日线只回答：

> 这段趋势的节奏怎样？是在持续创新高，还是新高越来越难出现了？

职责边界：
1. 新高日识别（当日收盘 > 过去 N 日最高价）
2. 新高序列计数（当前窗口内第几个新高日）
3. 距上一次新高的间距（新高间距放大 = 趋势活力衰竭早期信号）

日线**不负责**长期背景或中期顺逆归属。

### PAS 触发器 → 属于 `alpha/pas` 模块（不在 MALF 三层内）

**触发器和日线节奏是两件事，不能混淆。**

PAS 五触发器（BOF / PB / TST / CPB / BPB）回答的是：

> 在 MALF 背景 + 日线节奏的基础上，具体发生了哪种形态触发？

它们由 `alpha/pas` 模块独立检测，消费 `MalfContext`（月线 + 周线背景）作为输入，
不属于 MALF 模块的计算范围。

## 3. 为什么这样分层

| 层级 | 理由 |
|---|---|
| 月线做背景 | A 股暴涨暴跌容易切碎周线长期层；月线收盘线对大背景更稳定 |
| 周线做顺逆 | 周线波段数量约为月线的三倍，最适合回答"当前这段是主流还是逆流" |
| 日线做节奏（新高计数） | 日线最值钱的是节奏：新高日密集 = 趋势健康；新高日稀疏 = 趋势衰竭 |

类比（父系统原文）：
- **月线长期背景 = 季节**（决定你默认穿什么衣服）
- **周线当前关系 = 天气**（决定今天出门带不带伞）

## 4. 实用阅读顺序（冻结）

```text
1. 先看 long_background_2（由 monthly_state_8 收敛）
   → 长期背景是牛还是熊？

2. 再看 intermediate_role_2（由 weekly_flow 映射）
   → 当前中级波段是顺势还是逆势？

3. 确定 malf_context_4
   → 落在四格的哪一格？

4. 看生命周期三轴原始排位
   → 当前波段的 波幅/时间/新高频率 在历史同类中排第几？

5. 看总生命区间 lifecycle_rank_low / high
   → 三轴相加后，当前趋势生命走到哪里了？

6. 看日线节奏（新高计数）
   → 新高日还在密集出现，还是已经越来越难出现？

（如需看触发器，转 alpha/pas 模块，PAS 不在 MALF 阅读顺序内）
```

## 5. 对旧口径的正式纠偏（继承父系统冻结）

| 旧口径 | 当前正式口径 |
|---|---|
| `LONG` 以周线为主线 | 月线才是长期背景主线 |
| `INTERMEDIATE` 固定为日线 | 中期关系由周线承担 |
| 日线承担中级主对象 | 日线只做节奏现象（新高日计数）；PAS 触发器由 `alpha/pas` 独立承担 |
